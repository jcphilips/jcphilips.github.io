Core Principles

This document outlines the standardized methodologies for all CAD work. The goal is to establish a robust, scalable, and unambiguous system for file management, design structure, and collaboration. Adherence to these principles is critical for ensuring project integrity, manufacturing accuracy, and efficient workflow.

üìÑ One Component, One File

The foundational rule is that a single part file must only contain the geometry for a single, individual component. This ensures every manufacturable part has a unique digital identity, which is essential for accurate Bills of Materials (BOMs).

Case Study: Adding Fasteners

Incorrect: Inserting a clinch nut model into a sheet metal part file. The BOM will be wrong, and the nut won't be ordered.

Correct: The sheet metal part and the clinch nut are separate files. They are only brought together in an Assembly file.

üîó Assemblies Define Relationships

Assembly files have one purpose: to define the relationships between components using Joints. No part modeling should be done at the top level of an assembly. Components permanently joined before final assembly must be created as a Sub-Assembly.

Case Study: Handle with Inserts

Incorrect: Adding a handle, two inserts, and three joints to the main assembly. This creates unnecessary clutter.

Correct: Create a `Handle-Sub-Assembly.f3d` containing the handle and its inserts. Insert this single sub-assembly into the main product, simplifying the design.

Core Workflows

These workflows define the practical application of our design principles, ensuring a consistent process from initial design to final manufacturing.

Creating Sub-Assemblies

When a group of components are permanently joined together before the final assembly (e.g. with fasteners, welds, or adhesives), they must be created as a Sub-Assembly. This sub-assembly is then treated as a single "part" in the next level up, simplifying the main design and ensuring an accurate BOM.

Using Derived Designs for Manufacturing

The `Manufacturing` folder is a critical part of the release process. It ensures that the "design" models remain clean and separate from the "fabrication" models. Drawings should never be created from the primary design assembly. Instead, a Derived Part or Sub-Assembly must be created and placed in the `Manufacturing` folder.

In-Model Best Practices

A clean file is an understandable file. These practices must be adhered to within every design file to ensure clarity, ease of editing, and long-term maintainability.

1. Name Everything

All significant features in the browser tree and timeline must be given descriptive names. Rename `Sketch1` to `Profile-Main-Body` and `Body1` to `Lid`.

2. Group Timeline Features

Use the "Create Group" command to group related features in the timeline. Name the group "M4 Mounting Hole Feature Group." This makes the design history dramatically easier to navigate and debug.

3. Use Part Numbers

All custom components should ideally be given a unique part number at the root level. This number is the primary key for tracking a component through its entire lifecycle, from BOM creation to manufacturing and inventory.

Advanced Practices for Design & Manufacturing

Beyond basic modeling, these advanced techniques are crucial for creating robust, editable designs and clear, unambiguous manufacturing documentation.

‚öôÔ∏è Case Study: Parametric Design for COTS Parts

Scenario: A designer frequently uses pogo pin connectors, but each use case requires a different pin count and height. Modeling a new part for each variation is inefficient and clutters the component library.

Solution: A single, intelligent COTS part is created. In the Parameters dialog (`Modify > Change Parameters`), the designer defines key dimensions as "User Parameters."

pins_amount = 4

total_height = 10mm

The model's features are then driven by these parameters. The number of pins is controlled by the `pins_amount` parameter in a "Rectangular Pattern" feature, and the pin and holder bodies are controlled by the `total_height` parameter. Now, to create a 6-pin, 12mm tall version, the designer simply edits these two values. This parametric approach is ideal for creating configurable library parts, saving hours of repetitive modeling.

üè∑Ô∏è In-Model Documentation & Views

  • Named Sketches & Bodies: Essential for understanding a model's structure without digging through the timeline.
  • Part Numbers: Assign part numbers from your procurement system to the component's properties for seamless BOM integration.
  • Named Views (UPPERCASE): Create specific, named camera angles (e.g. `ISOMETRIC_MAIN`, `FLAT_PATTERN_VIEW`). Using these in drawings ensures all views are consistent, repeatable, and professional.
  • Named Storyboards: In the Animation workspace, create and name storyboards for different assembly or disassembly sequences. These are essential for generating clear exploded views in your drawings.

Case Study: The Multi-Stage Fabrication Process

Scenario: A complex sheet metal enclosure must be outsourced for fabrication. The process involves multiple distinct stages: laser cutting, bending, welding gussets, and installing two different sizes of clinch nuts. Communicating this sequence clearly is critical to avoid costly errors.

The Solution - A Two-Drawing Approach:

1. The Part Drawing (`...PRT...`): The designer first creates a drawing from a derived model of the main sheet metal part *only*. This drawing contains just the information for the initial fabrication steps: the flat pattern for laser cutting and the bend table/folded views for the press brake operator. It intentionally omits fasteners and gussets to avoid confusion.

2. The Sub-Assembly Drawing (`...SUB...`): A second drawing is created from the sub-assembly which includes the bent part, the gussets, and the clinch nuts. This drawing communicates the final assembly stages:

  • Named Storyboards created in the Animation workspace are used to generate clear Exploded Views. These views show exactly how the M3 and M4 clinch nuts are to be installed.
  • To clearly show clinch nuts on angled or obscured faces, specific **Named Views** like `INSTALL_NUTS_REAR_VIEW` are created. These custom camera angles are impossible to achieve with standard drawing views and are essential for unambiguous instructions.
  • To communicate the welding step, a view is created where the clinch nuts are **suppressed** (hidden). This focuses the welder's attention only on the gusset locations and the corresponding weld symbols.

This separation of drawings, enabled by deriving files and using advanced features like named storyboards and custom named views, is the only way to communicate a complex manufacturing sequence without ambiguity. It guides the fabricator through the process step-by-step, preventing errors and ensuring the final part is built exactly to specification.
